# Milestone 2: Text Processing — Implementation Plan

## Problem Statement

M1 delivered a correct, reliable interpreter with ~56 commands. M2 adds the heavyweight text processing tools that AI agents and shell scripts rely on for data manipulation: full `grep`, `sed`, `awk`, `jq`, `diff`, and remaining text utilities. These range from simple utilities (`tac`, `comm`) to full mini-languages (`awk`, `sed`).

## M1 Carryover

M1.13 (Case Statements) and M1.14 (Additional Core Commands) are both fully implemented in code but not marked as ✅ in `docs/guidebook/10-implementation-plan.md`. **Phase 0 fixes this documentation gap.** No code work remains from M1.

## Approach

- **Upgrade before build**: Start with upgrading existing `grep` (basic → full) rather than building from scratch.
- **Simple utilities first**: Build `comm`, `join`, `fmt`, `column`, `expand`/`unexpand`, `tac` as a warmup — these follow the established `VirtualCommand` pattern exactly.
- **Crate-backed where possible**: `jq` via `jaq-core` ecosystem, `diff` via `similar` crate. Don't reimplement what battle-tested crates solve.
- **Custom mini-interpreters for sed/awk**: These are fundamentally mini-languages that need lexer → parser → interpreter pipelines. Apply 80/20 scoping from the guidebook.
- **awk last**: It's the most complex command (associative arrays, functions, control flow). Everything else should be done before starting it.
- **Each phase is independently testable** and follows `cargo fmt && cargo clippy -- -D warnings && cargo test`.

## Dependencies

**Existing** (already in Cargo.toml):
- `regex = "1.12.3"` — used by grep, sed

**New** (to be added):
- `similar = "2.7.0"` — diff algorithm library for M2.5
- `jaq-core = "3.0.0-gamma"` — jq filter interpreter for M2.4
- `jaq-std = "3.0.0-gamma"` — jq standard library functions
- `jaq-json = "2.0.0-gamma"` — JSON value type for jaq
- `serde_json = "1"` — JSON parsing/serialization for jq command
- `rustyline = "17.0.2"` — readline for shell example (dev-dependency or optional)

> **Note on jaq versioning**: The jaq 3.x line is pre-release (gamma). If stability issues arise, fall back to the stable `jaq-interpret = "1.5.0"` + `jaq-parse = "1.0.3"` combination. Evaluate during Phase 5.

## Phase 1 — grep Full (M2.1)

> Upgrade the existing basic `GrepCommand` in `src/commands/text.rs`. The current implementation supports `-i`, `-v`, `-n`, `-c`, `-l`, `-F`. M2.1 adds the remaining flags to match real-world grep usage.

### 1a. Regex mode flags
- `-E` / `--extended-regexp` — extended regex (the `regex` crate uses ERE semantics natively, so this is a no-op flag, but must be accepted)
- `-G` / `--basic-regexp` — basic regex (default in POSIX grep). The `regex` crate does **not** support BRE natively (where `\(` is grouping, `+`/`?` are literal, etc.). Accept the flag but use ERE semantics regardless. Document this deviation — it matches the behavior of many non-POSIX grep implementations. Implement a `bre_to_ere()` translation helper in a shared utility location (reused by sed in Phase 4) to convert common BRE patterns: `\(` → `(`, `\)` → `)`, `\{` → `{`, `\}` → `}`. This provides best-effort BRE compatibility.
- `-P` / `--perl-regexp` — **Not fully supported.** The `regex` crate does not support PCRE's defining features (backreferences `\1`, lookahead/lookbehind `(?=...)`, possessive quantifiers). Accept the flag as an alias for `-E` and emit a one-time stderr warning: `grep: warning: -P is not fully supported, using extended regex`. This avoids silent mismatches.
- `-w` / `--word-regexp` — match whole words only (wrap pattern in `\b...\b`)
- `-x` / `--line-regexp` — match whole lines only (wrap pattern in `^...$`)

### 1b. Recursive search
- `-r` / `-R` / `--recursive` — recursively search directories via `ctx.fs.readdir()` + `ctx.fs.stat()`
- Only descend into directories, skip non-regular files
- When recursive, include filenames in output by default (like multi-file mode)
- `--include=GLOB` — only search files matching glob pattern
- `--exclude=GLOB` — skip files matching glob pattern

### 1c. Context lines
- `-A NUM` / `--after-context=NUM` — print NUM lines after each match
- `-B NUM` / `--before-context=NUM` — print NUM lines before each match
- `-C NUM` / `--context=NUM` — print NUM lines before and after each match
- Group separator: `--` between non-contiguous match groups

### 1d. Output control
- `-o` / `--only-matching` — print only the matched part of each line
- `-H` / `--with-filename` — always print filename (default when multiple files)
- `-h` / `--no-filename` — never print filename
- `-q` / `--quiet` / `--silent` — suppress output, exit 0 on first match
- `-m NUM` / `--max-count=NUM` — stop after NUM matches per file
- `-e PATTERN` — specify pattern (allows multiple patterns via repeated `-e`)
- `-f FILE` / `--file=FILE` — read patterns from a file (one per line)
- `-L` / `--files-without-match` — print names of files with no matches (inverse of `-l`)

### 1e. Refactor argument parsing
- The current character-by-character flag parser won't scale. Refactor to handle:
  - Long flags (`--recursive`, `--count`)
  - Flags with values (`-A 3`, `--context=5`, `-e PATTERN`)
  - Combined short flags (`-inl` = `-i -n -l`)
- Keep using manual parsing (no arg-parse crate) per project convention

### 1f. Tests
- `-E` with extended regex patterns (alternation `a|b`, groups `(abc)+`)
- `-G` with BRE-style patterns: verify best-effort BRE→ERE translation
- `-P` emits warning but still matches
- `-r` recursive search: create directory tree in VFS, search across files
- `-r` with `--include=*.txt` and `--exclude=*.log`
- `-A 2`, `-B 2`, `-C 2` context output with `--` separator lines
- `-o` only matching: `echo "foo123bar" | grep -o '[0-9]+'`
- `-w` word matching: `echo "cat concatenate" | grep -w cat` matches only "cat"
- `-e pat1 -e pat2` multiple patterns
- `-f patternfile` patterns from file
- `-m 1` max count
- `-q` quiet mode (no output, correct exit code)
- `-L` files without match (inverse of `-l`)
- Combined: `-rin` (recursive + case-insensitive + line numbers)
- Edge cases: empty pattern, binary-ish content, no matches (exit 1)

**Exit criteria**: Full grep flag coverage. All tests pass. `cargo clippy` clean.

## Phase 2 — diff (M2.5)

> New command in `src/commands/diff_cmd.rs`. Uses the `similar` crate for the diff algorithm.

### 2a. Add `similar` dependency
- Add `similar = "2.7.0"` to `Cargo.toml`
- Verify it compiles and basic API works

### 2b. Core diff implementation
- Compare two files (read from VFS)
- Default format: normal diff (ed-style: `NUMaNUM`, `NUMcNUM`, `NUMdNUM` with `<` / `>` prefixed lines)
- Algorithm: use `similar::TextDiff` for line-by-line comparison
- Handle stdin via `-` filename (use `ctx.stdin`)
- Exit codes: 0 (identical), 1 (different), 2 (error)

### 2c. Output formats
- `-u` / `--unified` — unified diff format (default context: 3 lines)
  - `--- file1` / `+++ file2` header
  - `@@ -start,count +start,count @@` hunk headers
  - ` ` (context), `-` (removed), `+` (added) prefixes
- `-c` / `--context` — context diff format (default context: 3 lines)
  - `*** file1` / `--- file2` header
  - `*** start,end ****` / `--- start,end ----` hunk headers
- `-U NUM` / `--unified=NUM` — unified with custom context lines
- `-C NUM` / `--context=NUM` — context with custom context lines

### 2d. Additional flags
- `-r` / `--recursive` — recursively compare directories
  - List files only in one dir, files only in the other, and diff common files
- `-q` / `--brief` — report only whether files differ
- `-s` / `--report-identical-files` — report when files are identical
- `-N` / `--new-file` — treat absent files as empty
- `-i` / `--ignore-case` — case-insensitive comparison
- `-w` / `--ignore-all-space` — ignore all whitespace
- `-b` / `--ignore-space-change` — ignore changes in amount of whitespace
- `-B` / `--ignore-blank-lines` — ignore blank line changes
- `--label LABEL` — use LABEL instead of filename in headers

### 2e. Tests
- Two identical files → exit 0, no output
- Two different files → exit 1, normal diff output
- `-u` unified format with correct headers and hunks
- `-c` context format
- `-r` recursive directory comparison
- `-q` brief mode
- `-i` case-insensitive diff
- `-w` ignore all whitespace
- `-b` ignore whitespace changes
- `-N` treat absent files as empty
- `-s` report identical files
- Single-line files, empty files, files with no trailing newline
- Stdin via `-`: `echo "hello" | diff - /file.txt`

**Exit criteria**: diff with all specified formats and flags. Tests pass. `cargo clippy` clean.

## Phase 3 — Remaining Text Commands (M2.6)

> Add to existing `src/commands/text.rs`. These are straightforward utilities following established patterns.

> **Note**: `yes` is already implemented in M1 (`src/commands/utils.rs`). Skip it.

### 3a. tac
- Reverse lines of file(s) or stdin (opposite of `cat`)
- `-s SEPARATOR` — use SEPARATOR instead of newline
- Read input → split → reverse → output

### 3b. comm
- Compare two sorted files line by line
- Three-column output: lines only in file1, only in file2, in both
- `-1` — suppress column 1 (lines unique to file1)
- `-2` — suppress column 2 (lines unique to file2)
- `-3` — suppress column 3 (lines common to both)
- `--check-order` — check input is sorted (default behavior)

### 3c. join
- Join lines of two sorted files on a common field
- `-t CHAR` — field separator (default: whitespace)
- `-j FIELD` — join on field FIELD for both files
- `-1 FIELD` / `-2 FIELD` — join field for file1/file2
- `-a FILENUM` — print unpairable lines from file FILENUM
- `-e STRING` — replace missing fields with STRING
- `-o FORMAT` — output format specification

### 3d. fmt
- Simple text formatter (reflow paragraphs)
- `-w WIDTH` — max line width (default: 75)
- `-s` — split long lines only, don't join short lines
- Preserve paragraph breaks (blank lines)

### 3e. column
- Columnate lists
- `-t` — create a table from delimited input
- `-s CHAR` — delimiter for `-t` mode (default: whitespace)
- `-o CHAR` — output separator (default: two spaces)
- Without `-t`: fill columns (newspaper-style)

### 3f. expand / unexpand
- `expand` — convert tabs to spaces
  - `-t N` — set tab stops to N spaces (default: 8)
  - `-t N1,N2,...` — set tab stops at specific positions
- `unexpand` — convert spaces to tabs
  - `-a` — convert all sequences of spaces (not just leading)
  - `-t N` — tab stop width

### 3g. Registration
- Register all new commands in `register_default_commands()` in `src/commands/mod.rs`

### 3h. Tests
- `tac`: reverse lines of multi-line input
- `comm`: two sorted files, suppress individual columns
- `join`: join two files on first field
- `fmt`: reflow paragraph to 40 characters
- `column -t`: tabulate delimited data
- `expand` / `unexpand`: tab ↔ space conversion
- Edge cases: empty input, single-line input, stdin vs file

## Phase 4 — sed (M2.2)

> New file `src/commands/sed.rs` containing the sed mini-interpreter. This is a custom implementation — no external crate. Uses the existing `regex` crate for pattern matching.

### 4a. sed data model
- Define internal types:
  ```
  SedAddress: LineNumber(usize) | Last | Regex(Regex) | Step(first, step)
  SedRange: Single(SedAddress) | Range(SedAddress, SedAddress)
  SedCommand: Substitute{regex, replacement, flags} | Delete | Print | Quit
              | Append(String) | Insert(String) | Change(String)
              | Label(String) | Branch(Option<String>) | BranchIfSubstituted(Option<String>)
              | HoldGet | HoldAppend | PatternGet | PatternAppend | Exchange
              | LineNumber | Next | NextAppend
  SedScript: Vec<(Option<SedRange>, SedCommand)>
  ```

### 4b. sed script parser
- Parse sed expressions (from `-e`, `-f FILE`, or script argument)
- **BRE→ERE translation**: sed uses BRE by default. Implement `bre_to_ere()` (shared with grep) to translate `\(` → `(`, `\)` → `)`, `\{` → `{`, `\}` → `}`, and make `+`, `?`, `|` literal by default. When `-E`/`-r` is passed, skip translation and use regex patterns as-is. This is critical — `s/\(.*\)/[\1]/` is one of the most common sed patterns and will silently break without BRE translation.
- Address parsing:
  - `N` — line number
  - `$` — last line
  - `/regex/` — regex match (with escaped delimiter support)
  - `N,M` — line range
  - `N~S` — step (every S-th line starting at N) — lower priority
- Command parsing:
  - `s/regex/replacement/flags` — substitution (flags: `g`, `i`, `p`, number)
    - Handle alternate delimiters: `s|old|new|`, `s#old#new#`
    - Replacement special chars: `&` (whole match), `\1`–`\9` (groups), `\n` (newline)
    - **Note**: The `regex` crate uses `$1`–`$9` for capture group references in replacements, not `\1`–`\9`. The sed replacement engine must translate `\1` → `${1}`, `\2` → `${2}`, etc. before passing to `regex::Regex::replace()`.
  - `d` — delete pattern space
  - `p` — print pattern space
  - `q` — quit (with optional exit code)
  - `a\text` / `a text` — append text after current line
  - `i\text` / `i text` — insert text before current line
  - `c\text` / `c text` — replace current line with text
  - `y/src/dst/` — transliterate characters
  - `=` — print current line number
  - `n` — output pattern space (unless `-n`), replace with next input line
  - `N` — append next input line to pattern space (with embedded newline)
  - `{ commands }` — command group (for address blocks)
- Multiple commands separated by `;` or newlines

### 4c. sed execution engine
- For each input line:
  1. Read line into pattern space
  2. For each command in script:
     - Check if address matches current line
     - If matched, execute command
  3. Unless `-n`, print pattern space
  4. Clear pattern space, read next line
- Track state: current line number, last-line flag, last substitution success (for `t`)

### 4d. Hold space operations
- `h` — copy pattern space to hold space
- `H` — append pattern space to hold space (with newline)
- `g` — copy hold space to pattern space
- `G` — append hold space to pattern space (with newline)
- `x` — exchange pattern and hold spaces
- These enable multi-line operations (e.g., joining lines, accumulating)

### 4e. Branching
- `:label` — define a label
- `b label` — branch (jump) to label; `b` alone jumps to end of script
- `t label` — branch if last `s///` succeeded; `t` alone jumps to end
- `T label` — branch if last `s///` did NOT succeed (GNU extension)

### 4f. Command-level flags
- `-n` / `--quiet` / `--silent` — suppress automatic printing
- `-e SCRIPT` — add script commands (can be repeated)
- `-f FILE` — read script from file (VFS)
- `-i` / `--in-place` — edit files in-place via VFS (`write_file` after processing)
- `-E` / `-r` — extended regex syntax
- Script from argument: `sed 's/old/new/' file`
- Script from `-e`: `sed -e 's/old/new/' -e 's/foo/bar/' file`

### 4g. Tests
- `s/old/new/` — basic substitution
- `s/old/new/g` — global substitution
- `s/old/new/i` — case-insensitive substitution
- `3d` — delete line 3
- `1,5p` — print lines 1–5
- `/pattern/d` — delete lines matching pattern
- Address ranges: `2,4s/a/b/g`
- `q` — quit after first line
- `a\appended`, `i\inserted`, `c\changed` — text insertion
- `y/abc/ABC/` — transliteration
- `=` — print line number
- `n` / `N` — next line operations
- `{ commands }` — command grouping with address: `2,4{ s/a/b/; s/c/d/; }`
- `-n` with `p` — selective output
- `-i` in-place editing (verify VFS file updated)
- Hold space: `sed -n 'H;${x;s/\n/ /g;p}'` — join all lines
- Branching: `sed ':a;N;$!ba;s/\n/ /g'` — join lines using labels
- Multiple `-e` expressions
- Alternate delimiters: `s|/path/old|/path/new|`
- Backreferences: `s/\(foo\)\(bar\)/\2\1/` (verifies BRE→ERE + `\1`→`$1` translation)
- Pipeline: `echo "hello world" | sed 's/world/earth/'`

**Exit criteria**: sed handles core commands, addresses, hold space, and branching. Tests pass. `cargo clippy` clean.

## Phase 5 — jq (M2.4) ✅

> New file `src/commands/jq_cmd.rs`. Wraps the `jaq-core` crate to provide jq-compatible JSON filtering.

### 5a. Add jaq dependencies
- Add to Cargo.toml:
  - `jaq-core = "3.0.0-gamma"`
  - `jaq-std = "3.0.0-gamma"`
  - `jaq-json = "2.0.0-gamma"`
  - `serde_json = "1"`
- If gamma versions cause issues, evaluate fallback to `jaq-interpret = "1.5.0"` + `jaq-parse = "1.0.3"` (note: API differs significantly between 1.x and 3.x)
- Verify compilation with a basic filter test
- **WASM check**: Verify `cargo check --target wasm32-unknown-unknown` still passes after adding jaq dependencies. If it fails, feature-gate the jq command behind `#[cfg(not(target_arch = "wasm32"))]` or find alternatives. This is a known risk — `serde_json` + jaq transitives may pull in std-dependent code.

### 5b. jq command wrapper
- Parse command-line arguments:
  - First non-flag argument is the filter expression
  - Remaining arguments are input files (read from VFS)
  - No files → read from `ctx.stdin`
- Use `jaq_core::load::Loader` + `Arena` for parsing:
  ```rust
  let defs = jaq_core::defs().chain(jaq_std::defs()).chain(jaq_json::defs());
  let funs = jaq_core::funs().chain(jaq_std::funs()).chain(jaq_json::funs());
  let loader = Loader::new(defs);
  let arena = Arena::default();
  let program = File { code: filter_str, path: () };
  let modules = loader.load(&arena, program)?;
  let filter = Compiler::default().with_funs(funs).compile(modules)?;
  ```
- For each input:
  - Parse JSON using `jaq_json::read::parse_single()` or `serde_json`
  - Run compiled filter via `filter.id.run((ctx, input))`
  - Collect output values

### 5c. Output formatting
- Default: pretty-printed JSON with syntax coloring disabled (since we return strings)
- `-r` / `--raw-output` — output raw strings without quotes (strings only; non-strings still JSON)
- `-c` / `--compact-output` — one-line JSON
- `-S` / `--sort-keys` — sort object keys
- `-j` / `--join-output` — like `-r` but no newline between outputs
- `-e` / `--exit-status` — exit 1 if last output is false or null

### 5d. Input options
- `-n` / `--null-input` — don't read input; `null` as input
- `-R` / `--raw-input` — read input as raw strings (one per line)
- `-s` / `--slurp` — read all inputs into an array
- `--arg NAME VALUE` — set `$NAME` to string VALUE
- `--argjson NAME VALUE` — set `$NAME` to parsed JSON VALUE

### 5e. Common filters to verify
These aren't things we implement (jaq handles them), but we must verify they work through our wrapper:
- `.field` — object field access
- `.field.subfield` — nested access
- `.[]` — iterate array/object values
- `.[N]` — array index
- `.field // "default"` — alternative operator
- `select(condition)` — filter
- `map(f)` — transform array elements
- `keys`, `values`, `length`, `type`, `has("key")`
- `to_entries`, `from_entries`
- `split("delim")`, `join("delim")`
- `test("regex")`, `match("regex")`
- `if-then-else`
- `reduce`
- `@base64`, `@uri`, `@html`, `@csv`, `@tsv`

### 5f. Error handling
- Invalid JSON input → stderr error, exit code 2
- Invalid filter syntax → stderr error, exit code 3
- Runtime filter error → stderr error, exit code 5
- No output produced → exit code 4 (with `-e`)
- Match jq exit code conventions where possible

### 5g. Tests
- `.name` on `{"name": "alice"}` → `"alice"`
- `.[] | .id` on array of objects
- `select(.age > 30)` filtering
- `map(.name)` transformation
- `-r` raw output: `echo '{"x":"hello"}' | jq -r '.x'` → `hello` (no quotes)
- `-c` compact: no whitespace in output
- `-s` slurp: multiple JSON values into array
- `--arg name val` variable injection
- `-n 'null'` null input
- Pipe chains: `.users | map(select(.active)) | length`
- Error cases: invalid JSON, invalid filter, missing field (null vs error)
- Large nested structures
- Multiple inputs (files and stdin)

**Exit criteria**: jq command works with common filters. Tests pass. `cargo clippy` clean.

## Phase 6 — awk (M2.3) ✅

> New directory `src/commands/awk/` containing the awk mini-interpreter. This is the most complex command — it's effectively a small programming language. Apply 80/20 scoping. Split into multiple files for maintainability:
>
> ```
> src/commands/awk/
> ├── mod.rs        # AwkCommand impl + VirtualCommand entry point
> ├── lexer.rs      # tokenizer
> ├── parser.rs     # AST types + parser
> └── runtime.rs    # interpreter, built-in functions, arrays
> ```

### Feature tiers (scope control)

| Tier | Features | Priority |
|------|----------|----------|
| **Must-have** | Field splitting (`$0`–`$NF`), `FS`/`OFS`/`RS`/`ORS`, `NR`/`FNR`/`NF`, `BEGIN`/`END`, `print`/`printf`, `if`/`while`/`for`, basic string funcs (`length`, `substr`, `index`, `split`, `sub`, `gsub`, `tolower`/`toupper`, `sprintf`), comparison operators, regex match (`~`, `!~`), `-F`, `-v`, associative arrays, `delete`, `for...in`, `break`/`continue`/`next`/`exit` | Core of Phase 6 |
| **Should-have** | `match()` (sets RSTART/RLENGTH), range patterns, assignment operators (`+=`, etc.), `int()`, ternary `?:`, `do...while`, `FILENAME`, implicit concatenation, `-f progfile` | Core of Phase 6 |
| **Nice-to-have** | Math funcs (`sqrt`, `sin`, `cos`, `atan2`, `exp`, `log`), `rand()`/`srand()`, `getline` (basic form only), `SUBSEP`/multi-dim arrays, `ARGC`/`ARGV` | Include if on schedule |
| **Defer** | User-defined functions, output redirection to files (`print > "file"`), pipe redirection (`print \| "cmd"`), complex `getline` forms (`getline < "file"`, `"cmd" \| getline`) | Future milestone |

### 6a. awk tokenizer
- Define tokens: `NUMBER`, `STRING`, `REGEX`, `IDENT`, `FIELD_REF` (`$N`), operators, keywords, punctuation
- Keywords: `BEGIN`, `END`, `if`, `else`, `while`, `for`, `do`, `break`, `continue`, `next`, `exit`, `return`, `function`, `in`, `delete`, `getline`, `print`, `printf`
- String literals with escape sequences (`\n`, `\t`, `\\`, etc.)
- Regex literals: `/pattern/`
- Handle concatenation (implicit operator in awk: `"a" "b"` = `"ab"`)

### 6b. awk parser
- Parse into AST:
  ```
  AwkProgram: Vec<AwkRule>
  AwkRule: { pattern: Option<AwkPattern>, action: AwkBlock }
  AwkPattern: Begin | End | Expression(Expr) | Regex(String) | Range(Expr, Expr)
  AwkBlock: Vec<AwkStatement>
  AwkStatement: Print{exprs, output_redirect} | Printf{format, exprs} | If{cond, then, else_}
              | While{cond, body} | For{init, cond, step, body} | ForIn{var, array, body}
              | DoWhile{body, cond} | Block(Vec<AwkStatement>) | Expression(Expr)
              | Break | Continue | Next | Exit(Option<Expr>) | Return(Option<Expr>)
              | Delete{array, index}
  Expr: Number(f64) | String(String) | FieldRef(Box<Expr>) | Var(String)
      | ArrayRef{name, index} | BinaryOp{op, left, right} | UnaryOp{op, expr}
      | Assign{target, value} | Ternary{cond, then, else_} | Match{expr, regex}
      | FuncCall{name, args} | Concat{left, right}
      | GetLine{var, source} | Regex(String)
      | InArray{value, array}
  ```

### 6c. awk runtime — core
- **Field splitting**:
  - Split input record on `FS` (default: whitespace, with special leading/trailing trim)
  - `$0` = entire record, `$1`...`$NF` = individual fields
  - Assigning to a field rebuilds `$0` using `OFS`
  - Assigning to `$0` re-splits fields
- **Built-in variables**:
  - `NR` — total records read so far
  - `FNR` — records read from current file
  - `NF` — number of fields in current record
  - `FS` — input field separator (default: `" "` = split on whitespace runs)
  - `OFS` — output field separator (default: `" "`)
  - `RS` — input record separator (default: `"\n"`)
  - `ORS` — output record separator (default: `"\n"`)
  - `FILENAME` — current input filename
  - `RSTART`, `RLENGTH` — set by `match()`
  - `SUBSEP` — subscript separator for multi-dimensional arrays (default: `"\034"`)
  - `ARGC`, `ARGV` — argument count and values
- **Command-line flags**:
  - `-F fs` — set field separator
  - `-v var=val` — set variable before execution
  - Program from argument: `awk 'program' files...`
  - Program from file: `-f progfile` (read from VFS)

### 6d. awk runtime — control flow & I/O
- **Pattern matching**:
  - `BEGIN { ... }` — execute before any input
  - `END { ... }` — execute after all input
  - `/regex/ { ... }` — match against `$0`
  - `expression { ... }` — truthy expression
  - `pattern1, pattern2 { ... }` — range (from first match to second match)
  - No pattern = match all records
- **Statements**:
  - `print expr1, expr2` — print with OFS between, ORS at end
  - `print` alone — print `$0 ORS`
  - **Output redirection**: `print > "file"` and `print >> "file"` are **deferred** (see tier table). All `print`/`printf` output goes to stdout for now. Pipe redirection (`print | "cmd"`) is also deferred as it requires the exec callback integration.
  - `printf format, expr1, expr2` — formatted output (C-style: `%d`, `%s`, `%f`, `%g`, `%x`, `%o`, `%c`, `%e`, `%%`)
  - `if (cond) stmt; else stmt`
  - `while (cond) stmt`
  - `for (init; cond; step) stmt`
  - `for (var in array) stmt`
  - `do stmt while (cond)`
  - `break`, `continue` — loop control
  - `next` — skip to next input record
  - `exit [code]` — exit with code

### 6e. awk runtime — operators & expressions
- Arithmetic: `+`, `-`, `*`, `/`, `%`, `^` (power)
- String concatenation: implicit (adjacent expressions)
- Comparison: `<`, `<=`, `==`, `!=`, `>=`, `>`
- Regex match: `~`, `!~`
- Logical: `&&`, `||`, `!`
- Ternary: `cond ? expr : expr`
- Assignment: `=`, `+=`, `-=`, `*=`, `/=`, `%=`, `^=`
- Increment/decrement: `++`, `--` (pre and post)
- Field reference: `$expr` (computed field reference)
- Unary minus, unary plus
- String-to-number coercion: awk's dual-type system (values are both string and number)
- Array membership: `(index in array)`

### 6f. awk runtime — built-in functions
- **String functions**:
  - `length(s)` — string length (or array length)
  - `substr(s, start [, len])` — substring
  - `index(s, target)` — find substring position
  - `split(s, array [, fs])` — split string into array
  - `sub(regex, replacement [, target])` — substitute first match
  - `gsub(regex, replacement [, target])` — substitute all matches
  - `match(s, regex)` — regex match, sets RSTART/RLENGTH
  - `sprintf(fmt, exprs...)` — formatted string
  - `tolower(s)` / `toupper(s)` — case conversion
- **Math functions**:
  - `int(x)` — truncate to integer
  - `sqrt(x)`, `sin(x)`, `cos(x)`, `atan2(y, x)`
  - `exp(x)`, `log(x)`
  - `rand()` — random 0–1
  - `srand([seed])` — seed random generator
- **I/O functions** (nice-to-have tier):
  - `getline` — basic form only: read next line into `$0`, return 1/0/-1. Complex forms (`getline var`, `getline < "file"`, `"cmd" | getline`) are deferred.

### 6g. awk runtime — associative arrays
- Arrays are associative (string-indexed hash maps)
- Auto-vivification: accessing `a["key"]` creates it
- `for (key in array)` iteration
- `delete array[key]` — remove element
- `delete array` — clear entire array
- Multi-dimensional: `a[i, j]` uses SUBSEP to create composite key
- `(key in array)` — membership test (returns 0 or 1)

### 6h. User-defined functions (deferred)
- `function name(params) { body }` — define function
- Local variables via extra parameters: `function f(a, b,    local1, local2)`
- `return expr` — return value from function
- Functions can be called recursively
- **Deferred**: Per the tier table, user-defined functions add significant parser and runtime complexity (separate scope management, recursion, local variable convention). Defer to a future milestone unless implementation is ahead of schedule.

### 6i. Tests
- **Basic**: `awk '{print $1}' file` — extract first field
- **Field separator**: `awk -F: '{print $1}' /etc/passwd`-style
- **Field assignment**: `awk '{$2 = "X"; print $0}'` — verify `$0` is rebuilt with OFS
- **Patterns**: `awk '/error/ {print}' log` — regex filter
- **BEGIN/END**: `awk 'BEGIN{sum=0} {sum+=$1} END{print sum}'` — sum column
- **Variables**: `awk -v threshold=10 '$1 > threshold'`
- **Uninitialized variables**: verify default to `""` (string) / `0` (numeric)
- **Expressions**: `awk '{print $1, $1*2}'` — arithmetic in output
- **Control flow**: `awk '{if ($1 > 10) print "big"; else print "small"}'`
- **printf**: `awk '{printf "%-20s %5d\n", $1, $2}'`
- **Arrays**: `awk '{count[$1]++} END{for(k in count) print k, count[k]}'` — word frequency
- **Array delete**: `awk '{a[$1]=1} END{delete a["x"]; for(k in a) print k}'`
- **String functions**: `awk '{print toupper($0)}'`, `split`, `sub`, `gsub`
- **Multi-file**: `awk '{print FILENAME, FNR, $0}' file1 file2`
- **Range pattern**: `awk '/start/,/end/ {print}'`
- **Pipe with other commands**: `cat file | awk '{print $2}' | sort -n`
- **No-action rule**: `awk '/pattern/' file` (implicit `{print}`)
- **Empty FS**: `awk -F '' '{print $1}'` — splits every character
- **Edge cases**: empty input, single-field records, empty fields, very long lines
- **NR vs FNR**: verify correct counts across multiple files

**Exit criteria**: awk handles the 80/20 subset (field splitting, patterns, actions, BEGIN/END, built-in variables, control flow, built-in functions, associative arrays). Tests pass. `cargo clippy` clean.

### 8b. Basic REPL loop (`examples/shell.rs`)

- Print non-zero exit codes: `[exit: N]`

### 8e. Output formatting

  - `echo 'echo hello' | cargo run --example shell` produces `hello`

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| awk complexity explosion | High | High | Strict tiered scope (must/should/nice/defer). Ship iteratively. Defer user-defined functions and output redirection. |
| sed edge cases (addresses, branching) | Medium | Medium | Start with s///, d, p. Add hold space and branching incrementally. Test each feature in isolation. |
| BRE→ERE regex translation bugs | Medium | Medium | Shared `bre_to_ere()` utility with thorough unit tests. Test against common real-world sed/grep BRE patterns. |
| jaq-core 3.x gamma instability | Medium | Medium | Have fallback plan to jaq-interpret 1.5.0. Pin exact version. Test thoroughly. |
| jaq-core WASM incompatibility | Medium | Medium | Check WASM target early in Phase 5a. Feature-gate if needed. |
| jaq-core API changes during M2 | Low | Medium | Pin to exact gamma version. Don't update mid-milestone. |
| regex crate limitations vs PCRE | Low | Low | Reject `-P` with warning. Document unsupported patterns. |
| diff performance on large files | Low | Medium | similar crate is well-optimized. Set output size limit via ExecutionLimits (M3). |
| sed `\1`→`$1` replacement translation | Medium | Medium | Test backreference patterns early. The translation is straightforward but easy to get wrong with escaping. |

## Notes

- All commands follow the `VirtualCommand` trait pattern (stateless, `Send + Sync`)
- All file access through `ctx.fs: &dyn VirtualFs` — never `std::fs`
- All commands support `--` flag terminator per Ch. 6 conventions
- File content from VFS is `Vec<u8>` — commands handle UTF-8 conversion with graceful error handling
- Error message format: `command: message` (e.g., `sed: invalid command: x`)
- sed and awk are custom interpreters — they don't delegate to external processes
- The `regex` crate is already a dependency; sed and grep share it
- `yes` is already implemented in M1, so M2.6 skips it
